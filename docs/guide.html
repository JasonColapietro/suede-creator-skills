Codex

Install into $CODEX_HOME/skills, or ~/.codex/skills when CODEX_HOME is not set.

git clone https://github.com/JasonColapietro/suede-creator-skills.git
cd suede-creator-skills
mkdir -p "${CODEX_HOME:-$HOME/.codex}/skills"
cp -R skills/suede-workflow-skills "${CODEX_HOME:-$HOME/.codex}/skills/"

Claude Code — manual

Install into a project .claude/skills folder or your user-level ~/.claude/skills folder.

git clone https://github.com/JasonColapietro/suede-creator-skills.git /tmp/suede-creator-skills
cd /path/to/your-project
mkdir -p .claude/skills
cp -R /tmp/suede-creator-skills/skills/suede-workflow-skills .claude/skills/

Suede workflow FAQ.

Short answers for builders, designers, agencies, creators, developers, and agents deciding whether the public Suede skill pack fits their workflow.

What are Suede Creator Skills?

Suede Creator Skills are free, MIT-licensed SKILL.md folders for Claude Code and OpenAI Codex. The pack covers multi-agent orchestration, code review with an A-F ship grade, CI ship-gates, AI evals, Suedify website restyling, design QA, conversion copy, SEO/AEO/AI EO audits, visibility grading, launch packaging, and a creator toolkit for campaigns, release prep, and rights evidence.

Which workflows do the skills support?

The skills support reference-site restyling, website redesign, copywriting, GitHub Pages polish, visibility and CTA grading, SEO/AEO/AI EO metadata, schema checks, public docs, launch packaging, code review with A-F grading, CI ship-gates, AI evals, and QA, plus artist campaign packaging and creator rights utilities.

When should I use Agent Teams?

Use Agent Teams when work is large, risky, public, cross-surface, release-bound, design-heavy, or likely to collide with user or other-agent WIP. It scouts first, splits safe lanes, keeps shared context fresh, reviews the result, verifies release truth, and records evidence.

Do the skills copy websites exactly?

No. Suedify studies layout, rhythm, typography, color roles, motion, and proof structure, then recreates the design grammar with target-owned content and assets.
