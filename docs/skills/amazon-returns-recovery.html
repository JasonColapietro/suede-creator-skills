Amazon Returns Recovery.

Amazon Returns Recovery scans an Amazon account's order and return history for restocking fees and short refunds the owner never noticed, reports every find, and — only after the owner confirms — drives Amazon's live chat to get the money waived and refunded. Documented real recoveries: $448.31 across three cases, including a full $372.69 refund on an item Amazon had already denied once, granted after the return window closed, with no return required.

What it covers

  • Phase 1 discovery is read-only: sweeps your-orders/search by category keyword (restocking fees concentrate on higher-value electronics), supplements with your-returns for recent activity, and expands each completed return's itemized refund breakdown to spot deducted fees.
  • Phase 2 stops and reports: item, order number, fee amount, ship-to, and first-party vs. marketplace — the account owner picks which to pursue. Some fees are legitimate, and the skill says so when one looks earned.
  • Phase 3 drives Amazon's live chat one order at a time, using a validated click-path through the contact hub, the bot qualifying questions, and escalation to a human associate.
  • Documents the popup-window gotcha: Amazon's chat opens via window.open() in a window Claude in Chrome cannot track — the reference includes the one-line JS patch that keeps the chat in the tracked tab.
  • Ground rules baked in: state only true facts, never invent prior contact or return reasons, confirm the exact refund amount and timeline in writing before ending the chat.
  • Phase 4 reports per dispute: amount recovered, refund method, stated ETA, and a running session total.

When to use it

  • The user mentions Amazon returns, restocking fees, or a refund that looked short — even without the word "restocking."
  • Checking whether a past return was fully refunded, or whether Amazon is still holding money across an account's history.
  • Disputing a specific fee or short refund the user already knows about, with the case drafted and driven through live chat.
  • Requires the Claude in Chrome browser extension connected and signed into the target Amazon account.
  • Not for: marketplace third-party disputes that need the A-to-z Guarantee process, bank chargebacks, or price-protection claims (an unvalidated stretch goal, off by default).
