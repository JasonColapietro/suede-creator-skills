A-F code grading

Suede Code Grader.

Suede Code Grader gives diffs, branches, PRs, MCP servers, plugins, APIs, public-site changes, app surfaces, and release candidates an A-F ship-risk grade with a plain explanation of why the grade landed there.

What it grades

  • Correctness, edge cases, error paths, routing, and regression risk.
  • Security, permissions, secrets, payment, wallet, injection, and data exposure risk.
  • Data and state behavior across schemas, migrations, jobs, caches, webhooks, retries, and state transitions.
  • Suede truth: public copy, rights, provenance, registry, royalty, licensing, and agent-commerce claims.
  • UX, release behavior, tests, verification, deploy readiness, and rollback clarity.

How to use it

Call the grader when the user wants a fast but serious A-F code score with the reason why. Use full Suede Code Review when the work needs prioritized findings, fix briefs, and deeper regression hunting.

In major agent-team work, run the code grader after builder output and before release lock. Pair it with adversarial review and release verification when the change is public, risky, or release-bound.

Boundary

The grade is a working review, not a certification or audited security result. The grader does not invent tests, screenshots, live checks, deploy status, or public claim evidence.
