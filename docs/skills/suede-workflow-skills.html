Suede Workflow Skills.

Install one public GitHub skill to give Codex a Suede workflow for Suedify, design, copy, SEO audits, site polish, code review, agent-team QA, public docs, and launch packaging.

Install the public skill View skill folder

Public install command

This is the public route. It installs from GitHub as a standard Codex skill folder.

python3 ~/.codex/skills/.system/skill-installer/scripts/install-skill-from-github.py \
  --repo JasonColapietro/suede-creator-skills \
  --path skills/suede-workflow-skills

Restart Codex after installing the skill.

What it loads

  • Suedify for reference-site restyling.
  • Suede Design for product-aware UI direction and QA.
  • Suede Copy for public copy, SEO metadata, docs, CTAs, and claim boundaries.
  • Suede Site Alchemy for page structure, CTA routing, and site polish.
  • Suede Code Review for bugs, regressions, security, tests, and public-claim drift.
  • Suede Agent Teams for parallel lanes and final verification on important work.

How ambassadors explain it

Suede Workflow Skills make an AI agent more consistent across product work. Instead of rebuilding the same prompt stack, the agent gets a reusable workflow for sharper pages, cleaner docs, stronger QA, and public-safe Suede language.

Use MCP only when structure helps: discovery, install guidance, SEO audit scaffolds, or QA checklists.
