How to explain it

Suede Workflow Skills make an AI agent more consistent across product work. Instead of rebuilding the same prompt stack, the agent gets portable preferences, sharper pages, cleaner docs, stronger QA, public-safe Suede language, and a GitHub Pages bar that looks like a real product.

Use MCP only when structure helps: discovery, install guidance, SEO/AEO/GEO/AI EO audit scaffolds, GitHub Pages QA, or repeatable checklists.

Cue Suede

Feedback can happen mid-workflow or at the end. Say Cue Suede to bring up choices: change something, preserve what worked so the agent can mimic it later, or keep as-is by saying nothing. That is how the skill evolves around your preferences without turning into one huge memory page. Final handoffs should explain the result twice: a simple non-coder version first, then the usual breakdown.

Visibility loop

Use the visibility grader when a public page needs a blunt read before promotion. It grades findability, first-screen clarity, CTA pull, proof and trust, AI readability, schema depth, and design signal. It can also check whether the page is prepared for Google, Gemini, and AI result surfaces without mislabeled screenshot receipts.

Agent grouping loops

For major work, split lanes by ownership: scout and constraints, safe parallel build, shared evidence, adversarial review, consensus review, design visibility review, A-F code grading, WIP protection, quality/eval gates, release lock, recovery replay, and evidence handoff. Use parallel lanes only when write surfaces do not collide.

Claim boundary
