# Suede Campaign In A Box

Use this skill to turn scattered release ideas into a campaign an artist can
actually execute.

## Package

1. Identify the release, artist state, audience, platforms, timeline, assets,
   and one action the campaign must earn.
2. Build the campaign phases:
   - seed;
   - tease;
   - announce;
   - release week;
   - post-drop;
   - fan proof;
   - catalog afterlife.
3. Include copy, content prompts, fan actions, proof links, and asset requests.
4. Define the next build when the campaign needs a website, drop, unlock,
   collector action, agent, or app workflow.
5. Keep claims grounded. Do not invent streams, press, cosigns, placements, or
   legal clearance.

## Public Copy Gate

Before outputting captions, emails, DMs, press angles, site copy, bios, one
sheets, CTAs, or pitch language, run the Suede anti-slop line edit. Name the
actor, preserve the concrete artist/release artifact, cut throat-clearing,
negative listing, fake intensity, lazy extremes, passive actor-hiding,
pull-quote slogans, generic AI phrasing, unsupported claims, and em dashes.
