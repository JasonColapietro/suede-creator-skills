# Suede Public Claim Check

Use this skill before public Suede copy ships.

## Check

1. Identify every claim about legal status, ownership, payouts, royalties,
   registry writes, distribution, partner access, traction, users, automation,
   or install availability.
2. Mark each claim as verified, unverified, too broad, or unsafe.
3. Replace unsafe wording with concrete facts, links, commands, artifacts, or
   narrower language.
4. Make local-only details explicit when they appear.
5. Add a claim boundary near rights, money, licensing, or release language.

## Output

```text
Unsafe claim:
Why it is risky:
Safer replacement:
Proof needed:
Public-ready copy:
Ship gate: ship | ship-with-caveats | hold
```
